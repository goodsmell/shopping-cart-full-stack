# 장바구니 FE 2단계 요구사항

## 1. 기능 요구사항

### 1.1 장바구니 상품 목록

- 장바구니 API를 호출하여 상품 데이터를 불러온다.
- 진입 시 전체 선택이 기본값이다.
- 상품 선택 여부는 새로고침 시에도 유지된다.

### 1.2 상품 선택

- 상품 선택/해제 시 결제 금액을 동적으로 변경한다.
- 결제 금액이 10만원 이상일 경우 배송비는 무료이다.

### 1.3 수량 변경

- 장바구니 상품의 수량을 변경할 수 있다.
- 수량은 최대 99개까지 가능하다.

### 1.4 상품 삭제

- 장바구니에 담긴 상품을 제거할 수 있다.
- 상품명은 최대 100자이다.

---

## 2. 기술 요구사항

### 2.1 상태 관리

- `useState`, `useReducer`, Context API를 사용하여 상태를 관리한다.
- 결제 금액, 배송비 등의 파생 상태는 동적으로 계산한다.

### 2.2 데이터 페칭 훅

- 서버 API 통신 결과를 관리하는 훅을 직접 구현한다.
- 반환값에는 data, loading, error가 포함되어야 한다.
- 상용 라이브러리를 사용하지 않고 직접 구현한다.

### 2.3 API

- 1단계에서 만든 API를 그대로 사용한다.
- API 요청 중에는 로딩 UI를 표시한다.
- API 요청 중 에러가 발생한 경우 에러 메시지를 사용자에게 표시한다.

### 2.4 테스트

- Jest, React Testing Library를 사용하여 주요 기능을 테스트한다.
- 비동기가 포함된 테스트는 MSW를 활용한다.

### 2.5 코드 퀄리티

- 컴포넌트와 비즈니스 로직을 분리하여 재사용 가능하게 한다.
- 세부 구현을 드러내는 props 네이밍을 사용하지 않는다. (DIP 적용)

### 2.6 라이브러리

- 스타일링은 Emotion을 사용한다.
- 명시된 라이브러리 외에는 사용하지 않고 직접 구현한다.

## 3. 테스트 목록

- API 테스트
  - 장바구니 목록 불러오기
  - 장바구니 삭제
  - 수량 변경
- 장바구니에 담긴 상품의 수를 계산한다.
- 선택한 상품의 개수를 계산한다.
- 선택한 상품의 주문금액을 계산한다.
- 배송비 추가 여부를 결정한다
- 총 결제 금액을 계산한다 (주문금액 + 배송비)
